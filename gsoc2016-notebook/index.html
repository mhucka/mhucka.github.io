How do you create the skeleton of an Express application?
This page describes it: http://expressjs.com/en/starter/installing.html
Assuming you’ve already installed Node.js, create a directory to hold your application, and make that your working directory.
$ mkdir myapp
$ cd myapp
Use the npm init command to create a package.json file for your application. For more information on how package.json works, see Specifics of npm’s package.json handling.
$ npm init
This command prompts you for a number of things, such as the name and version of your application. For now, you can simply hit RETURN to accept the defaults for most of them, with the following exception:
entry point: (index.js)
Enter app.js, or whatever you want the name of the main file to be. If you want it to be index.js, hit RETURN to accept the suggested default file name.
Now install Express in the app directory and save it in the dependencies list. For example:
$ npm install express --save

Node modules installed with the --save option are added to the dependencies list in the package.json file. Afterwards, running npm install in the app directory will automatically install modules in the dependencies list.

What’s the basic hello world?
This page has an example: http://expressjs.com/en/starter/hello-world.html
var express = require('express');
var app = express();

app.get('/', function (req, res) {
res.send('Hello World!');
});

app.listen(3000, function () {
console.log('Example app listening on port 3000!');
});
I saved the above to a file called helloworld.js.
When I run node helloworld.js in the directory where it’s located, and open a browser window to localhost:3000, it does indeed work.
Note that it doesn’t send back HTML — the words “hello world” are just plain text. From this we learn that the res.send() command above does not do anything about HTML formatting.
What’s the use of the --save option?
Some instructions tell you to use --save. What does that do?
npm install express --save
This makes it write a line into the package.json file to indicate that the installed package (in this example, express) is a dependency.
How does the express generator work?
Get express generator like this:
npm install express-generator -g
Using -g instead of --save here makes it install it as a general command in your shell path, rather than a dependency of the application you're writing. This makes sense because this is a tool that you would use for all kinds of applications and not just a specific one.

What is express.static(...) about?
express.static is a built-in middleware function. It is used to indicate where to find static assets that are to be served. "Pass the name of the directory that contains the static assets to the express.static middleware function to start serving the files directly. For example, use the following code to serve images, CSS files, and JavaScript files in a directory named public:"
http://expressjs.com/en/starter/static-files.html
app.use(express.static('public'));
To use multiple static assets directories, call the express.static middleware function multiple times:
app.use(express.static('public'));
app.use(express.static('files'));
Express looks up the files in the order in which you set the static directories with the express.static middleware function.
To create a virtual path prefix (where the path does not actually exist in the file system) for files that are served by the express.static function, specify a mount path for the static directory, as shown below:
app.use('/static', express.static('public'));
This will let you refer to URLs with the given prefix:
http://localhost:3000/static/images/kitten.jpg
http://localhost:3000/static/css/style.css
Note that the directory is relative to where you started the node process. It's better to use the following form:
app.use('/static', express.static(__dirname + '/public'));
What is "routing" about?
Official docs are pretty complete and concise:
2016-06-expressjs.com-en-guide-routing
http://expressjs.com/en/guide/routing.html
It's how an application determines how to respond to requests from clients to a particular endpoint, which is to say a given URI and specific HTTP request method (GET, PUT, etc.)
In express, each route can have one or more handler functions. These are executed when the route is matched. The definitions take this form:
app.METHOD(PATH, HANDLER)
app is an instance of express.
METHOD is an HTTP request method, in lowercase.
PATH is a path on the server.
HANDLER is the function executed when the route is matched.
Here's a simple example of handling GET requests to the root of the application:
var express = require('express');
var app = express();

app.get('/', function(req, res) {
res.send('hello world');
});
Route paths can be strings, string patterns, or regular expressions.
Query strings are not part of the route path.
Express supports the following routing methods that correspond to HTTP methods: get, post, put, head, delete, options, trace, copy, lock, mkcol, move, purge, propfind, proppatch, unlock, report, mkactivity, checkout, merge, m-search, notify, subscribe, unsubscribe, patch, search, and connect.
There is a special routing method, app.all(), which is not derived from any HTTP method. This method is used for loading middleware functions at a path for all request methods.
You can have more than one function handle the same route, by specifying the "next" object in the arguments and calling next() in the body:
app.get('/example/b', function (req, res, next) {
console.log('response will be sent by the next function ...');
next();
}, function (req, res) {
res.send('Hello from B!');
});
The methods on the response object (res) in the following table can send a response to the client, and terminate the request-response cycle. If none of these methods are called from a route handler, the client request will be left hanging.
Method
Description
res.download()
Prompt a file to be downloaded.
res.end()
End the response process.
res.json()
Send a JSON response.
res.jsonp()
Send a JSON response with JSONP support.
res.redirect()
Redirect a request.
res.render()
Render a view template.
res.send()
Send a response of various types.
res.sendFile()
Send a file as an octet stream.
res.sendStatus()
Set the response status code and send its string representation as the response body.
What is express.Router?
It's a so-called "mini-app" that lets you create modular, mountable route handlers.
Example from http://expressjs.com/en/guide/routing.html
Suppose this is in the file birds.js:
var express = require('express');
var router = express.Router();

// middleware that is specific to this router
router.use(function timeLog(req, res, next) {
console.log('Time: ', Date.now());
next();
});
// define the home page route
router.get('/', function(req, res) {
res.send('Birds home page');
});
// define the about route
router.get('/about', function(req, res) {
res.send('About birds');
});

module.exports = router;
Then load the router module in the app:
var birds = require('./birds');
...
app.use('/birds', birds);
The app will be able to handle requests to /birds and /birds/about, and will call the timeLog middleware function defined above.
How do you render plain HTML?
You don't need to render it per se; you can either send a single file using res.sendFile() or use the express.static() middleware function (see above) for serving static assets from a directory.
How do you set up error handlers?
They are the same as other Express middleware, but take 4 args, using the signature (err, req, res, next):
app.use(function(err, req, res, next) {
console.error(err.stack);
res.status(500).send('Something broke!');
});
How do you use Handlebars.Js?
npm install express-handlebars --save
Note: The engine name has to be the same as the file extension
var express = require('express');
var exphbs = require('express-handlebars');

app.set('views', path.join(__dirname, 'views/'));

app.engine('hbs', exphbs({defaultLayout: 'main', extname: '.hbs'}));
app.set('view engine', 'hbs');

app.get('/', function (req, res) {
res.render('index');
});
How are layouts handled in express-handlebars?
The default dir for layouts is "views/layouts/"
A layout is simply a Handlebars template with a {{{body}}} placeholder. Usually it will be an HTML page wrapper into which views will be rendered.
https://github.com/ericf/express-handlebars
What are handlebars helpers?
They are "functions that can be registered with Handlebars and can be called within a template. Helpers can be used for transforming output, iterating over data, etc. To keep with the spirit of logic-less templates, helpers are the place where logic should be defined."
https://github.com/ericf/express-handlebars
Handlebars ships with some built-in helpers, such as: with, if, each, etc

How do you substitute values in mustache templates?
Call res.render with a 2nd argument consisting of a json object, with the entries corresponding to the names of the templates.
app.get('/random', function(req, res) {
res.render('random', { random: randomText.getText() });
});
Can you use custom _id types in Mongoose?
Yes. Define the value type of _id in the schema; don't let it default, or else Mongoose wil assume it's an ObjectId, and then you will see errors about casts when you try to retrieve documents by id values.
