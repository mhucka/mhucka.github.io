Notebook usage notes
Notes in a notebook are in chronological order as much as possible.
Level 1 rows are dates. This way, in OmniOutliner, it’s convenient to be able to see all the dates in the left-hand content pane.
I use headings to indicate changes in topics or lines of thought
The heading style is “Heading 1”.
Notes under a heading often involve the same topic — whatever I happen to be working on or thinking about. When I change directions, I try to introduce a heading to indicate the change. This makes reading notes later.
Handling clippings
PDFs, clippings and other pasted-in material must be inserted in the “Clippings” column for a given row.
Use the notes to indicate the source, or provide a URL.
Even though pasted-in material should have a note that indicates the URL, the items themselves should be linked to their sources.
Flag column
The flag column has a few symbols that can be used to annotate notes. This is for reminders to myself to pay extra attention to something, or to job my memory.

What is the data type of rdf:ID in Biopax documents?
The type is listed as RDF/XML ID. Looking around for a definition, after some effort, I finally traced things down to the page at http://www.w3.org/TR/REC-rdf-syntax/#rdf-id wherein it states that it is "An attribute ·string-value· matching any legal [XML-NS] token NCName". Now looking up the XML data type NCName, we find it in http://www.w3.org/TR/REC-xml-names/#NT-NCName that it states
An XML Name, minus the ":"
That's then defined in terms of something else, the XML Name type, which looks like this:
[4] NameStartChar ::= ":" | [A-Z] | "_" | [a-z] | [#xC0-#xD6] | [#xD8-#xF6] | [#xF8-#x2FF] | [#x370-#x37D] | [#x37F-#x1FFF] | [#x200C-#x200D] | [#x2070-#x218F] | [#x2C00-#x2FEF] | [#x3001-#xD7FF] | [#xF900-#xFDCF] | [#xFDF0-#xFFFD] | [#x10000-#xEFFFF]

[4a] NameChar ::= NameStartChar | "-" | "." | [0-9] | #xB7 | [#x0300-#x036F] | [#x203F-#x2040]

[5] Name ::= NameStartChar (NameChar)*
Practically speaking, this means the allowed characters includes underscore, letters, digits, and also period "." and hyphen "-", but not colon.

identiifiers:
Biopax has rdf:ID, which seems closest to sbml 'id' attributes
the names need to be munged
Andreas points out that ordinary Java functions for detecting whether something is a letter or number may not be appropriate because they allow more characters than the strict subset of letters and numbers allowed by SBML (e.g., they consider Unicode accented characters to be letters whereas SBML doesn't). Suggests using something like this (from the SBVC code, I guess?):
public static boolean isLetter(char c) { return ((c >= 97) && (c <= 122)) || ((c >= 65) && (c <= 90)); }
Biopax also has a displayName field, which could be used to set the sbml 'name' attribute.
If there is no displayName for a given Biopax thing, then perhaps the rdf:ID could be reused for the SBML name as a fall-back. In that case, also, it would not need to be munged, because SBML name attributes can have almost any character, unlike the SBML id.
But this is a question of conventions. Another option is to not set the SBML name attribute if no displayName is given in the Biopax object.
(Now thinking about it further, maybe that's the best option.)
reactions:
there are subclasses of biopax interactions for control and conversion
they have left & right
the left and right may not necessarily be an indication of direction -- it may not be that the "left" must be a reactant and the "right" be a product, if the interaction actually represents a reversible reaction
Ah, it turns out that biopax has a "conversionDirection" element. Could this be what determines whether a reaction is reversible or not? Does that have to be used to interpret biopax interactions and convert them to sbml reactions?
